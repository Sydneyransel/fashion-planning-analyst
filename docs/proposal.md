## Reflection

This Planning Analyst role at SKIMS directly applies the skills covered in this course, which focuses on SQL for relational databases, working with APIs, and building web-based applications that interact with back-end databases. The posting explicitly requires SQL proficiency and the ability to extract insights from data to support demand forecasting and inventory planning — both of which map to the query language, database manipulation, and API work covered throughout the semester. To prove I can do this work, I'll build an end-to-end pipeline that pulls Google Trends data via a Python API client, loads it into Snowflake, transforms it through a dbt star schema using SQL, and surfaces category-level demand trends in a deployed Streamlit dashboard — a live artifact that mirrors what a Planning Analyst would build internally. This same project transfers cleanly to similar roles such as a Demand Planner at a DTC apparel brand, an Inventory Analytics Analyst at a retailer like Revolve or Nordstrom, or a Retail Insights Analyst at a fashion holding company like PVH or Tapestry.
